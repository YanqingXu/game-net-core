## Intended release position

This preview extends the frozen Reactor/TCP core with the first installed game
network foundation components. It is an integration and contract preview, not
a production-readiness declaration.

Installed CMake targets intended for this preview:

- `GameNet::core`
- `GameNet::protocol`
- `GameNet::transport`
- `GameNet::game_session`
- `GameNet::game_logic`
- `GameNet::broadcast`

`gamenet_game_server_pipeline_demo` remains a non-installed example.
`gamenet_phase4_benchmark` remains an opt-in, non-installed, non-CTest
engineering executable.

## Proposed highlights

### Packet framing

- Unsigned 32-bit big-endian length-prefixed framing.
- Explicit payload, buffered-byte, frame-count, and processed-frame-byte
  limits.
- `BudgetExhausted`/`needsContinuation` fairness contract and ring-buffered
  unread storage.
- Deterministic segmented round-trip coverage plus an optional differential
  libFuzzer harness.

### Transport lifetime boundary

- Narrow TCP endpoint adapter with stable transport identity.
- Copyable, non-owning `EventLoopExecutor` scheduling capability.
- Explicit `WrongThread`, `OwnerUnavailable`, and `Closed` results.
- Cross-thread `isOpen()` snapshot without a cached dereferenceable loop
  pointer.
- Final-drain semantics: work admitted before EventLoop shutdown may complete
  owner-only endpoint operations, while admission after closure is rejected.

### Session and logic foundation

- Management-loop-owned player and transport indexes.
- Explicit duplicate-login and transport-id/endpoint collision behavior.
- Manager public results expose read-only `PlayerSession` views, preventing
  external rebind/state mutation behind the indexes.
- Bounded cross-thread command admission, fixed-size tick drains, one-shot
  lifecycle, and stop-time backlog accounting.

### Three-loop Pipeline example

- Real TCP flow from I/O-owned framing through management-owned authentication
  and a separately owned LogicLoop back to the endpoint owner loop.
- Explicit `Unauthenticated -> Authenticating -> Online -> Closing` state.
- Bounded pending input while authentication is outstanding.
- Same-write `AUTH + command` ordering and revocable callback admission.
- Callback-state ownership does not preserve execution permission after
  revocation; every loop/timer/output/endpoint side effect rechecks the shared
  alive state.
- Configurable duplicate-login policy with deterministic rejection and queued
  pre-auth input discard coverage.
- Safe synchronous `stop()` re-entry from the Logic stage when management and
  logic share one EventLoop, plus endpoint-observer revocation before send.

### Broadcast

- Management-loop routing with shared payload ownership and endpoint
  deduplication.
- Hard/soft fanout and byte policies with reason-specific metrics.
- Move-only, router-created plans and executor-based per-owner dispatch.
- Bounded task fanout/bytes, multi-loop real TCP delivery, and versioned Phase 4
  latency/memory benchmark output.
- Eight real TCP disconnect/reconnect cycles prove stale endpoints are not
  retained in subsequent route plans.

### IOCP lifecycle corrections

- Acceptor completion storage remains alive until canceled completion dequeue.
- Connector ConnectEx timeout/stop cancellation retains operation and Channel
  lifetime until a late completion packet is consumed, including the
  `CancelIoEx` completion-won race.

## Evidence contracts implemented locally

These contracts define what a future authoritative run must produce; their
existence is not proof that such a run has occurred.

- Main `ci` has six producers (`linux-cmake`, `linux-asan-ubsan`, `linux-tsan`,
  `linux-release`, `windows-msvc`, and `windows-msvc-release`) plus one
  aggregation-only `ci-evidence-set` job. Each producer writes
  `gamenet.ci_evidence.v1` with a canonical
  `<prefix>-<job>-<sha>-<run_id>-<run_attempt>` artifact identity. The
  `gamenet.ci_evidence_set.v1` aggregate accepts exactly those six jobs from one
  run/attempt/SHA, rehashes every declared file, and verifies exact inventory
  and JUnit selections. It is not a seventh platform producer.
- `long-soak` derives the 61 threading and 8 Pipeline/Broadcast selections from
  the 85-test inventory. `gamenet.ctest_repeat_evidence.v1` requires every
  selected test to pass exactly 50 times, rejects missing/extra/non-passing
  results, records total and per-test executions, and hashes the inventory and
  raw log. The canonical long-soak artifact carries both raw logs and both
  structured summaries.
- `core-benchmark` has Linux epoll and Windows IOCP producers plus an
  aggregation-only pair gate. Each platform preserves the three raw scenario
  JSON files, semantic manifest, toolchain, and `gamenet.ci_evidence.v1`
  wrapper. `gamenet.phase4_benchmark_pair_evidence.v1` requires the exact pair,
  common run/SHA identity, correct runner/platform/backend, identical scenario
  parameters and order, and matching hashes.

## Known limitations

- Public Phase 4 APIs are preview-grade and may change before a stable release;
  no source or ABI compatibility promise is made yet.
- The Pipeline is an example, not an installed production shutdown coordinator.
  It revokes callback/output admission and coordinates LogicLoop teardown, but
  it does not expose a public asynchronous completion/future proving every
  server close callback has completed when `stop()` returns.
- LogicLoop is intentionally one-shot. Its owner must preserve the object
  through timer and handler/output/metric callback completion; restart after
  stop is rejected.
- Transport `send()` and `close()` remain endpoint-owner-loop operations.
  Cross-thread callers must use `ownerExecutor()` and handle rejected admission.
- PacketFramer is caller-owned and intentionally not thread-safe. Its configured
  bounds must be selected for the deployment and protocol.
- Broadcast does not compute AOI, room, team, or world membership. Local real-
  TCP reconnect cycles and bounded high-fanout contracts pass; final same-SHA
  remote soak/benchmark evidence is still pending.
- Benchmark numbers are raw comparison snapshots, not correctness thresholds,
  capacity guarantees, or production sizing claims. Framing currently reports
  throughput rather than allocator-level allocation counts, and process RSS is
  sampled only as a coarse process-level observation.
- TLS, HTTP, WebSocket, RPC, coroutine bridges, UDP, and KCP remain deferred or
  experimental and are not included in this preview.
- No current-SHA remote TSan, sanitizer-backed libFuzzer, repeat-50 long-soak,
  six-producer CI aggregate, paired benchmark, or GitHub Release evidence
  exists yet.

## Preview API and downstream upgrade notes

Downstream consumers preparing for this preview should account for these
changes:

1. Use `find_package(GameNetCore REQUIRED)` and link only the installed targets
   needed by the application.
2. Replace cached/raw endpoint owner-loop use with `ownerExecutor()` and handle
   executor admission failure plus `EndpointResult::OwnerUnavailable`.
3. Treat `SessionManager::authenticate()` and `findBy*()` session pointers as
   `shared_ptr<const PlayerSession>`; all lifecycle mutation remains manager
   owned.
4. Handle PacketFramer budget and fault statuses explicitly. Continue buffered
   work with `push({})` only when `needsContinuation` is true.
5. Treat LogicLoop as one-shot and consume the returned stop/drop summary.
6. Obtain `BroadcastPlan` only from `BroadcastRouter::route()` and move it into
   `BroadcastDispatcher::dispatch()`.
7. Do not depend on Pipeline headers or targets as an installed framework API.

The following preview surfaces should be called out as unstable in the formal
release: `EventLoopExecutor`, PacketFramer budget/result types,
`TransportEndpoint`, SessionManager result/view types, LogicLoop lifecycle and
metrics, Broadcast plan/reason/metric types, and the example-only Pipeline
options/stage observer.
