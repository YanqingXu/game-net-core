# v0.1.0-core-preview Historical Record

`v0.1.0-core-preview` is an annotated source tag, not a GitHub Release. This
record documents the evidence and limitations that existed when the tag was
created; it does not retroactively claim release assets that were never
published.

## Identity and scope

- Tag: `v0.1.0-core-preview`
- Peeled commit: `c4818d4b3956c85830e04d4a1f32df4ad701d453`
- Package version: `GameNetCore 0.1.0`
- Installed target: `GameNet::core`
- Scope: the Reactor/TCP Core through Phase 3.5; Phase 4 protocol, transport,
  session, logic, pipeline, and broadcast APIs are not part of this tag.

## Known limitations

- This is Preview-grade Core, not a production-readiness declaration or an ABI
  stability promise.
- TSan covered the `threading` label selection rather than the complete CTest
  inventory.
- Long-soak and benchmarks were manual evidence workflows, not push gates.
- Windows Release was represented by local/package and manual benchmark
  evidence rather than a required main CI job.
- Benchmark values are raw snapshots without regression thresholds or capacity
  guarantees.
- HTTP, WebSocket, RPC, TLS, UDP/KCP, coroutine, and game-foundation layers are
  outside the tag.

## Downstream use

Consumers should require the exact preview package when reproducibility matters:

```cmake
find_package(GameNetCore 0.1.0 EXACT REQUIRED)
target_link_libraries(my_target PRIVATE GameNet::core)
```

The `0.2.0` Phase 4 preparation adds independent upper-layer targets and changes
preview contracts; consumers should review its upgrade notes instead of
assuming source or ABI compatibility.
